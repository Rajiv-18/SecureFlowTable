================================================================================
                         THE THREE TESTS
================================================================================

Test 1: BASELINE (No Attack)
  Purpose: Establish normal network behavior
  Expected: ~5 flows, 1-2ms latency, 10% CPU usage
  Duration: 10 minutes
  
Test 2: UNDER ATTACK (Without Defense)
  Purpose: Show how vulnerable networks behave
  Expected: 1000+ flows, 50-200ms latency, 90% CPU
  Duration: 15 minutes
  
Test 3: UNDER ATTACK (With Defense)
  Purpose: Show the defense working
  Expected: 500 flows, 2-10ms latency, 30% CPU
  Duration: 15 minutes

Comparison: Defense reduces impact by 75% - flows, latency, CPU all improved!

================================================================================
                        PROJECT COMPONENTS
================================================================================

TOPOLOGY (Network Creation)
  File: topology/simple_topology.py
  Does: Creates virtual network with 2 hosts and 1 switch
  Learn: How Mininet and OpenFlow work
  
CONTROLLERS (Network Intelligence) - CHOOSE ONE:
  File 1: controller/basic_controller.py
  Does: Simple learning controller, shows vulnerability
  Use: For baseline and attack (no defense) tests
  
  File 2: controller/defense_controller.py  
  Does: Advanced controller with 3 defense mechanisms
  Use: For attack (with defense) test
  Learn: Rate limiting, anomaly detection, flow management

ATTACK (Simulate the Problem)
  File: attack/attack_script.py
  Does: Sends many unique packets to fill flow table
  Learn: How attacks work, packet generation with Scapy

MONITORING (Collect Measurements)
  File: monitoring/monitoring.py
  Does: Measures flows, latency, CPU during tests
  Learn: Performance monitoring and metrics collection

UTILITIES (Helper Functions)
  File: utils/sdn_utils.py
  Does: Common operations (query flows, parse logs, etc.)
  Learn: OpenFlow utilities and common patterns
